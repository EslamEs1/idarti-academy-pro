# Tasks: Student-Teacher Assignment & Academic Supervision Frontend

**Input**: Design documents from `/specs/011-student-teacher-assignments/`
**Prerequisites**: plan.md (required), spec.md (required for user stories), research.md, data-model.md, contracts/, quickstart.md

**Tests**: This is a static-frontend prototype. The spec does NOT request automated tests. Validation is via manual visual review + grep-based audits documented in quickstart.md §3 "Success criteria validation table" (12 SC rows) + §5 "Constitutional 12-gate Definition of Done".

**Organization**: Tasks are grouped by user story:
- **US1** (P1 🎯 MVP) — Admin assigns a student to a teacher with the correct pricing and earning preview → ships `student-teacher-assignments.html` + `create-student-teacher-assignment.html`.
- **US2** (P2) — Admin transfers a student between teachers while preserving every historical link → ships `student-teacher-assignment-details.html` + `transfer-student-teacher.html`.
- **US3** (P2) — Teacher reviews their assigned students and follows up on at-risk ones → ships `my-assigned-students.html` + `assigned-student-details.html`.
- **US4** (P3) — Coordinator monitors the academy's academic supervisors → ships `academic-supervisors.html`.

The anchor assignment `ASN-2024-1184` (عبد الرحمن مؤمن × الأستاذ أحمد) is the spine: it threads US1 (a row on the assignments list), US2 (the admin-details page), US3 (the teacher-view page + a row on the teacher's roster), and US4 (a supervision row). The `<!-- SHARED_ASN_HEADER -->` block authored in US2 is reused byte-identically in US3 (SC-004 `diff` check), so US3's `assigned-student-details.html` depends on US2's `student-teacher-assignment-details.html` for that block. The sidebar-propagation phase (Phase 7) is intentionally last — the 7 new pages embed the *updated* sidebar from the locked snippets (Foundational T005/T006), so propagating the same insert to prior-spec pages does not block any user story.

## Format: `[ID] [P?] [Story] Description`

- **[P]**: Can run in parallel (different files, no dependencies on incomplete tasks)
- **[Story]**: Which user story this task belongs to (US1 / US2 / US3 / US4)
- Setup / Foundational / Sidebar-Propagation / Polish tasks have NO story label
- Include exact file paths in descriptions

- Static HTML pages live under `pages/admin/` and `pages/teacher/` at the repository root.
- Shared CSS bundle: `assets/css/output.css` (rebuilt once during Polish).

- Spec design docs: `specs/011-student-teacher-assignments/`.
