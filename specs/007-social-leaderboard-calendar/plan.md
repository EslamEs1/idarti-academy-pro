# Implementation Plan: Social Hub, Leaderboard, and Calendar Frontend

**Branch**: `007-social-leaderboard-calendar` | **Date**: 2026-05-09 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/007-social-leaderboard-calendar/spec.md`

## Summary

Build seven new pages — four on the student side (`social-hub`, `post-details`, `leaderboard`, `calendar`), two on the admin side (`social-hub`, `create-post`), and one on the teacher side (`calendar`) — that fulfill every prior-spec promise about the academy's safe-community + fair-recognition + clear-scheduling layer. The student dashboard sidebar already links to `social-hub.html` (line 114), `leaderboard.html` (line 122), and `calendar.html` (line 130) — plus an in-body CTA on line 474 ("عرض الجدول"). The admin dashboard sidebar already links to `social-hub.html` (line 183). The teacher dashboard already links to `calendar.html` from both the sidebar (line 103) and an in-body CTA (line 212, "جدولي اليوم"). Constitution v1.0.1 enumerates "Social Hub" + "Leaderboard" + "Calendar" in the Student sidebar, "Social Hub" in the Admin sidebar, and "Calendar" in the Teacher sidebar. None of the seven destinations exist today — every link 404s on every existing page that embeds those sidebars (≥ 25 student pages + 1 admin page + 1 teacher page). Spec 007 ships them all.

The seven pages are:

- `pages/student/social-hub.html` — calm, scannable feed of academy-curated posts. 7 post cards (one per type from the locked taxonomy: إعلان عام / تهنئة / إنجاز طالب / تحدي الأسبوع / نصيحة تعليمية / تذكير اختبار / منشور دورة). NO student composer (FR-011). Curated positive-only 5-reaction quintet (FR-012 — 👍 / ⭐ / ❤️ / 🤲 / 👏). 8-chip type-filter strip + moderation-explainer banner ("التعليقات تخضع للمراجعة...") at the top. "تهنئة" + "إنجاز طالب" cards anonymize the celebrated student to first-name + last-initial ("عبد الرحمن م." / "سارة م.") per the Q3-clarification privacy posture (FR-015 extension + FR-025 + FR-062).
- `pages/student/post-details.html` — single-post deep-read. Full hero image + 4-6 paragraph Arabic body, the 5-reaction quintet (identical order to social-hub per FR-012), ≥ 4 anonymized comments with explicit moderation status badges (≥ 1 "تم الاعتماد" + ≥ 1 "في انتظار المراجعة" per FR-013), comment form with "إرسال للمراجعة" button + 24-hour SLA helper text (per the Q4-clarification — FR-014), and ≥ 3 related-post mini-cards.
- `pages/student/leaderboard.html` — motivational ranking surface. 6-month chip strip with أبريل 2026 highlighted; "بطاقة ترتيبك" personal-rank hero card pulled out above the tabs (cream/parchment aesthetic from Spec 006's monthly-report header — FR-034); 6 tabs via existing main.js tabs handler ("الترتيب العام" / "الحضور" / "الواجبات" / "الأكثر تطوراً" / "تقدّم القرآن" / "المشاركة"); per-tab top-10 table with the persona's row visually highlighted (rank #6 in الترتيب العام, rank #2 in الأكثر تطوراً — FR-036); the "ⓘ كيف نحسب النقاط؟" 6-line accordion explainer (per the Q3-clarification — FR-038a — uses native `<details>`/`<summary>`, no JS required); the "تقدّم القرآن" tab is enrollment-scoped (Quran-enrolled only — per Q2 clarification — FR-035 extension); closing motivational panel with `monthly-report.html` CTA.
- `pages/student/calendar.html` — unified events surface. CSS-only month grid (7 columns, Saturday-first RTL, 5-6 rows depending on month, today's cell — 9 مايو 2026 — visually highlighted); 6-chip type-filter strip ("الكل" + 5 type chips: حصة مباشرة / تسليم واجب / اختبار / إصدار تقرير / إعلان أكاديمي); event-list panel with ≥ 8 chronological rows covering all 4 statuses (قادم / مكتمل / فائت / اليوم); each "عرض" CTA links to the correct prior-spec destination (live-session-details / assignment-details / exam-details / monthly-report / post-details — FR-044). The "أضف للتقويم الشخصي" cluster (Apple/Google/Outlook visual stubs) appears ONLY here (FR-019).
- `pages/admin/social-hub.html` — admin moderation workspace. h1 "إدارة المجتمع التعليمي" + primary "إنشاء منشور جديد" CTA → `create-post.html`. 4 moderation-summary cards (تعليقات في انتظار المراجعة / منشورات منشورة / منشورات مجدولة / مسوّدات). Filter row (3 dropdowns + search). Posts table with ≥ 8 rows covering all 5 visible statuses (منشور / مسودة / مجدول / غير منشور / محذوف — FR-051), per-row 5-icon-button action cluster (👁 عرض / ✏ تعديل / ↗ نشر / ⛔ إلغاء نشر / 🗑 حذف — visual only — FR-050), prototype-notice at the top of the table.
- `pages/admin/create-post.html` — admin authoring form. 7-option post-type select; title input + 6-row textarea; 4-option audience radio group with conditional course `<select>` for "دورة محددة"; file-upload visual placeholder ("ارفع صورة" — drop-zone aesthetic, no real upload); 2-option scheduling radio group with date+time visual inputs rendered unconditionally; 3-button footer cluster (نشر المنشور / حفظ كمسودة / إلغاء link → `social-hub.html`). 100% `<label for>` pairing per FR-008 / FR-060.
- `pages/teacher/calendar.html` — teacher's unified events surface. Same month-grid + event-list structure as the student calendar (shared visual language) but renders teacher-relevant data per Q2 clarification: own classes, grading deadlines (تسليم واجب → `homework-review.html`, NOT `assignment-details.html` — FR-069), invigilation slots, student-report submission deadlines (إصدار تقرير → `student-reports.html` — FR-069), teacher-targeted announcements. Includes a teacher-only "خلاصة هذا الأسبوع" workload-summary strip below the subtitle (FR-068) and EXCLUDES the personal-calendar export cluster (FR-070).

All seven pages reuse the existing app shells verbatim. Four student pages embed the existing 14-entry student sidebar with the appropriate active-entry per page (`social-hub.html` + `post-details.html` → "Social Hub"; `leaderboard.html` → "Leaderboard"; `calendar.html` → "Calendar"). Two admin pages embed the existing admin sidebar with "Social Hub" active. The teacher page embeds the existing teacher sidebar with "Calendar" active. All header chrome (search / bell / profile dropdown) is identical to the prior-spec baseline. **Zero new JavaScript** ships under this feature — the leaderboard tabs reuse the existing `[role="tablist"]` handler from Spec 002/003/004/006; the explainer accordion uses native `<details>`/`<summary>` HTML5 elements (which require no JS — see research.md §R5).

The two clarifications resolved during `/speckit-specify` plus the three additional clarifications resolved during `/speckit-clarify` lock the central design decisions:

- **Q3 — Achievement-post privacy (clarify)**: Anonymize "تهنئة" / "إنجاز طالب" post subjects to first-name + last-initial format (e.g., "عبد الرحمن م.") matching the leaderboard pseudonym convention (FR-039) and the spirit of the comments anonymization rule (FR-015). - **Q4 — Quran Progress tab scope (clarify)**: Enrollment-scoped — the "تقدّم القرآن" tab renders only students currently enrolled in any Quran course, with a scope caption ("هذا الترتيب يشمل الطلاب المسجّلين في دورات القرآن — نقيس التقدّم في المسار، ليس الالتحاق به."). The sibling سارة (Arabic L1) MUST NOT appear in this tab. Other 5 tabs remain academy-wide.

- **Q7 — Reaction-tap behavior (clarify)**: Non-interactive — tapping a reaction button MUST NOT open any modal, tooltip, or popover revealing reactor identities. ## Technical Context

**Language/Version**: HTML5; CSS via TailwindCSS 3.4.x (already configured in repo); ES2020 vanilla JavaScript (zero new code added under this feature — `assets/js/main.js` remains at the Spec 001/005/006 baseline of 68 lines). No new dependencies introduced.
**Primary Dependencies**: Existing `tailwindcss` devDependency + `@tailwindcss/forms` + `autoprefixer` + Tajawal Google Fonts CDN — all carried over from Specs 001-006. No new libraries; no charting library (the leaderboard surfaces values as numbers + horizontal progress bars, not charts).
**Storage**: N/A — static prototype. All "data" (post bodies, comments, reactions counts, leaderboard rows, calendar events, admin posts table, teacher workload summary) is hard-coded Arabic sample content embedded directly in each HTML page.
**Testing**: Manual visual review against the 24 success criteria in `spec.md` (SC-001 → SC-024) plus per-page acceptance scenarios in user stories US1-US7. Browser-based responsive validation at 375 / 768 / 1024 / 1280 px on each of the seven pages. Cross-spec reconciliation grep checks (SC-005 + SC-006 + SC-011 + SC-013 + SC-014 + SC-015 + SC-016 + SC-017 + SC-022 + SC-023 + SC-024) run as part of the Polish phase. No automated test framework in scope.
**Target Platform**: Modern evergreen browsers (latest Chrome, Edge, Firefox, Safari) on desktop, tablet, and mobile. Pages MUST open via `file://` URLs without a dev server.
**Project Type**: Continuation of the single static frontend project at the repository root. No new top-level directories. The `pages/student/` directory grows by four files (`social-hub.html`, `post-details.html`, `leaderboard.html`, `calendar.html`); the `pages/admin/` directory grows by two (`social-hub.html`, `create-post.html`); the `pages/teacher/` directory grows by one (`calendar.html`).
**Performance Goals**: Each page reaches first meaningful paint within ~1 s on a typical mid-range laptop opening from disk; the seven new pages add ≤ 7 KB to the compiled `output.css` after `npm run build:css`, keeping the project under the 80 KB CSS budget locked in Spec 001 (current size at the end of Spec 006 = 63,191 bytes → headroom ≈ 17 KB before the 80 KB ceiling).
**Constraints**: No new JavaScript handlers — only the four sanctioned in Spec 001/006 carry over (sidebar drawer / dropdown / modal / tabs). The leaderboard 6-tabs uses the existing `[role="tablist"]` handler verbatim. The leaderboard "كيف نحسب النقاط؟" explainer uses native HTML5 `<details>`/`<summary>` (zero JS needed — confirmed in research.md §R5). The calendar's CSS-only month grid uses Tailwind `grid-cols-7` + RTL layout (no SVG, no JS). No `href="#"` placeholder links (FR-007); no Eastern Arabic-Indic digits (FR-006); no JS-rendered content (FR-002); no thumbs-down / laughing / anger reactions anywhere (FR-012); no student-side post composer (FR-011); no real moderation / publish / delete / upload / share action (FR-003). All Save / Publish / Schedule / Upload / Delete / Share buttons are `<button type="button">` visual stubs. The admin "حذف" action keeps the row visible at opacity-60 + line-through (FR-050) — no row-removal actually occurs.
**Scale/Scope**: 7 pages (4 student + 2 admin + 1 teacher); 0 new sidebar entries on any role (every entry referenced is constitutionally enumerated and already present in prior-spec sidebars); 0 header changes; 7 entity types modeled in static HTML across the 7 pages; ~180 unique sample-data records (7 post cards × ≥ 6 attributes each on social-hub; 1 hero post + 4 comments + 3 related-posts on post-details; 6 dimensions × 10-row table + 1 personal card on leaderboard; ~20 events × 7 attributes on each calendar; 8+ posts × 7 attributes + 4 summary cards on admin social-hub; full create-post form with 7+4+2 enumerations + sample pre-fills). Persona continuity from Specs 002-006 (عبد الرحمن مؤمن / سارة مؤمن / parent / 3 teachers); admin author voice "إدارة الأكاديمية" newly named here. Calendar "today" anchor: 9 مايو 2026.
