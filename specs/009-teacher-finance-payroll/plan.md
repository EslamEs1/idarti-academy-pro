The "no-broken-links + multi-currency-visibility + read-only-for-teacher + dual-lens-settlement + regulatory-transparency" design discipline is the spec's distinguishing pattern. Eight structural levers enforce it:

- **Zero prior-spec edits** — every existing teacher and admin sidebar already points to the new file paths (validated by repository grep before authoring); the implementation only writes new files.
- **Zero `href="#"`** — SC-004 requires a clean grep across all 8 new pages; the «تواصل مع الإدارة» footer caption uses non-link text, the FR-086 «تعديل بيانات الحساب» affordance uses a static-disabled caption.
- **Read-only-for-teacher** — SC-010 requires zero `<input type="number">` matches across the 3 teacher-side pages; only filter/search inputs (sanctioned by Constitution V) are permitted.
- **Two-views-one-entity** — STL-2026-04-TCH-0042 renders byte-identical financial figures on `teacher/settlement-details.html` and `admin/teacher-settlement-details.html`; the salary rule (30/60/50/100 + 20%) renders consistently on `admin/teacher-salary-rules.html` and as the side panel on `teacher/earnings.html`.
- **Multi-currency without column-mixing** — `admin/teacher-finance.html` separates "إجمالي المستحقات بالريال" and "إجمالي المستحقات بالجنيه"; per-row currency badges; no SAR/EGP cross-summing.
- **Arabic-vs-Foreign rate distinction visibility** — SC-007 requires ≥ 12 contexts each of "ناطق بالعربية" + "غير ناطق بالعربية" across the 8 files (the spec already exceeds this with 23 + 23).
- **Advance-settlement traceability** — FR-095 + SC-006 require the same -200 + -250 = -450 ريال figure to appear consistently across teacher/settlement-details, admin/teacher-settlement-details, admin/teacher-finance row, admin/teacher-advances rows, and teacher/advances rows; cross-references via "ADV-2026-0012" + "ADV-2026-0023" must each appear ≥ 6 times.
- **Sanctioned-modal action confirmation** — Q1 mandates 3 confirmation modals on admin/teacher-settlement-details.html using the existing main.js modal handler; zero new JS.

Sixteen success criteria grep-validate the discipline (SC-001 page-existence + sidebar-active; SC-004 zero broken links; SC-007 Arabic-vs-Foreign distinction visibility; SC-008 EGP three-surface presence; SC-010 read-only enforcement; SC-014 zero new JS lines; SC-015 two-views-one-entity arithmetic).
