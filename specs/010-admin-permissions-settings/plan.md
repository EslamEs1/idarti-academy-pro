# Implementation Plan: Admin Permissions, Roles, Platform Settings & Pricing Rules Frontend

**Branch**: `010-admin-permissions-settings` | **Date**: 2026-05-10 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/010-admin-permissions-settings/spec.md`

## Summary

Build ten new admin pages — `roles-permissions`, `role-details`, `create-role`, `users`, `user-details`, `platform-settings`, `pricing-rules`, `student-type-settings`, `notification-settings`, `audit-log` — that close the **administrative-control layer** of the platform: the back-of-house surfaces governing who-does-what across the entire academy. A repository-wide grep before authoring confirmed that the two *entry-point* pages of this layer are already linked from every prior-spec admin sidebar (`roles-permissions.html` 18 refs, `platform-settings.html` 34 refs across `pages/admin/*.html`); the other 8 pages are reachable from those entry points and from each other. Spec 010 ships all ten pages with **zero prior-spec edits**, mirroring the Spec 008 + Spec 009 discipline.

The ten pages are:

- `pages/admin/roles-permissions.html` — academy-wide governance hub. 4 summary tiles (الأدوار الفعّالة 7 / الأدوار الموقوفة 1 / إجمالي المستخدمين 14 / الصلاحيات المعرّفة 19×7) + 7 active role cards (Admin / Finance Manager / Teacher / Parent / Student / Support / Operations) + 1 archived role card (مساعد إداري — موقوفة + «إعادة تفعيل» CTA) + page-header «+ إنشاء دور جديد» CTA → `create-role.html`. Each card carries users-count, permissions-count summary (e.g., "18 من 19 وحدة — مسموح كامل" or "6 من 19 وحدة — مالي فقط"), status badge, and «عرض/تعديل» CTA → `role-details.html`.
- `pages/admin/role-details.html` — single-role deep-read for the Finance Manager role (the dual-lens demo role). 19-module × 7-action permission matrix (133 cells) + 3-glyph legend at top (✓ مسموح / ✕ محجوب / – غير منطبق) + metadata strip below matrix (الدور / المستخدمون المعيّنون 3 + «عرض المستخدمين» link / آخر تعديل بواسطة محمد بن عبدالعزيز / الحالة) + «حفظ التعديلات» / «إلغاء» action buttons. ## Technical Context

**Language/Version**: HTML5; CSS via TailwindCSS 3.4.x (already configured in repo); ES2020 vanilla JavaScript (zero new code added under this feature — `assets/js/main.js` remains at the Spec 001/005-009 baseline of 68 lines, including the modal handler reused for the Q3 user-details modals). No new dependencies introduced.
**Primary Dependencies**: Existing `tailwindcss` devDependency + `@tailwindcss/forms` + `autoprefixer` + Tajawal Google Fonts CDN — all carried over from Specs 001-009. No new libraries; no charting library; no icon library beyond inline SVG (Lucide-style glyphs already used since Spec 001).
**Storage**: N/A — static prototype. All "data" (role cards, permission-matrix cells, user table rows, pricing rules, notification toggle states, audit events, platform-settings defaults, modal copy) is hard-coded Arabic-first sample content embedded directly in each HTML page.
**Testing**: Manual visual review against the 17 success criteria in `spec.md` (SC-001 → SC-017) plus per-page acceptance scenarios in user stories US1-US3 (15 scenarios). Browser-based responsive validation at 375 / 768 / 1024 / 1280 px on each of the ten pages (with special attention to role-details.html's 19×7 = 133-cell matrix on tablet — must be horizontally-scrollable via `overflow-x-auto` + `min-w-[900px]`). Cross-spec reconciliation grep checks (zero prior-spec edits / zero `href="#"` / zero new JS / 4 prior-spec identifiers in audit-log / 6+ contexts each for ناطق-بالعربية / غير-ناطق-بالعربية / two-views-one-entity matrix consistency) run as part of the Polish phase. Modal interaction smoke-test on user-details.html. No automated test framework in scope.
**Target Platform**: Modern evergreen browsers (latest Chrome, Edge, Firefox, Safari) on desktop, tablet, and mobile. Pages MUST open via `file://` URLs without a dev server.
**Project Type**: Continuation of the single static frontend project at the repository root. No new top-level directories. The `pages/admin/` directory grows by exactly 10 files (`roles-permissions.html`, `role-details.html`, `create-role.html`, `users.html`, `user-details.html`, `platform-settings.html`, `pricing-rules.html`, `student-type-settings.html`, `notification-settings.html`, `audit-log.html`).
**Performance Goals**: Each page reaches first meaningful paint within ~1 s on a typical mid-range laptop opening from disk. The new pages reference Tailwind utilities largely already present in the Spec 008/009 baseline; the JIT scan will pick up any new utility (e.g., `min-w-[900px]` for the 19×7 matrix horizontal-scroll container, `grid-cols-[repeat(7,minmax(60px,1fr))]` for the matrix grid, additional toggle-switch utilities for the notification-settings 28-toggle grid) automatically.
